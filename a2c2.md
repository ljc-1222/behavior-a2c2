# A2C2 Dataset

這份文件說明 A2C2 資料集是什麼、資料如何取得、如何儲存，以及如何用
`a2c2_create_dataset.py` 下載或重建資料。

## 快速下載

公開資料集已上傳到 Hugging Face：

```text
https://huggingface.co/datasets/ljc-1222/a2c2_dataset
```

在專案環境中執行一支程式即可下載完整資料：

```bash
python a2c2_create_dataset.py
```

預設會把資料下載到：

```text
./a2c2_dataset
```

如果要指定位置：

```bash
python a2c2_create_dataset.py --download-root /path/to/a2c2_dataset
```

常用下載選項：

```text
--repo-id ljc-1222/a2c2_dataset   Hugging Face dataset repo
--download-root PATH              本機下載位置
--revision REV                    指定 branch / tag / commit
--force-download                  強制重新下載
--local-files-only                只使用本機 HF cache
--max-workers N                   下載並行 worker 數
```

若環境沒有 `huggingface_hub`，先安裝：

```bash
python -m pip install huggingface_hub
```

## A2C2 是什麼

A2C2 的目標是在 frozen base policy 的 action chunk 上訓練一個 correction
head，而不是重新訓練整個 base policy。對 source frame `t` 和 chunk offset
`k`：

```text
a_exec[t+k] = a_base[t, k] + delta_a[t, k]
delta_a[t, k] = correction_head(o[t+k], a_base[t, k], tau[k], z[t], language)
```

其中：

- `a_base[t, k]` 是 COMET/OpenPI PI0.5 在 frame `t` 產生的 action chunk。
- `z[t]` 是同一個 base policy 對 frame `t` 的內部 latent 表示。
- `o[t+k]` 和 expert action 來自 BEHAVIOR demonstration。
- `delta_a[t, k] = expert_action[t+k] - a_base[t, k]` 是 correction head 的訓練目標。

## 資料來源

這份資料集來自 BEHAVIOR-1K challenge demonstrations：

```text
BEHAVIOR-1K/OmniGibson/datasets/2025-challenge-demos
```

目前發布的 variant 是：

```text
tidying_bedroom_pi05-b1kpt50-cs32_h32_v1
```

產生流程：

1. 選取 BEHAVIOR task `tidying_bedroom`，`task_index=18`。
2. 保留原始 LeRobot/BEHAVIOR parquet、metadata、videos、annotations。
3. 使用 OpenPI/COMET checkpoint `pi05-b1kpt50-cs32` 和 config `pi05_b1k-base`。
4. 對每個 source frame 執行 base policy inference，存出 `a2c2.base_action_chunk`。
5. 依照 episode 尾端長度產生 `a2c2.valid_action_mask`。
6. 從同一個 COMET/OpenPI prefix forward 擷取 mask-pooled PaliGemma hidden state 作為 `a2c2.base_policy_z`。

## z latent 從哪裡來

`z` 不是 action denoising loop 的輸出，而是 COMET/OpenPI PI0.5 prefix forward
的最後 hidden states 經 mask pooling 後的向量。概念上對應：

```python
prefix_tokens, prefix_mask, prefix_ar_mask = module.embed_prefix(observation)
prefix_attn_mask = make_attn_mask(prefix_mask, prefix_ar_mask)
positions = jnp.cumsum(prefix_mask, axis=1) - 1
(prefix_out, _), _ = module.PaliGemma.llm(
    [prefix_tokens, None],
    mask=prefix_attn_mask,
    positions=positions,
)
z = sum(prefix_out * prefix_mask) / sum(prefix_mask)
```

因此 `z` 包含 image tokens、prompt tokens、以及 PI0.5 離散化 state tokens 經
PaliGemma contextual transformer 後的摘要。它和 action chunk 使用同一個
checkpoint、task prompt、input transform。

## 儲存格式

下載後的資料根目錄如下：

```text
a2c2_dataset/
  README.md
  tidying_bedroom_pi05-b1kpt50-cs32_h32_v1/
    manifest.json
    data/task-0018/episode_XXXXXXXX.parquet
    latent/data/task-0018/episode_XXXXXXXX.parquet
    latent/meta/manifest.json
    meta/info.json
    meta/tasks.jsonl
    meta/episodes.jsonl
    meta/episodes_stats.jsonl
    meta/episodes/task-0018/episode_XXXXXXXX.json
    videos/task-0018/<video_key>/episode_XXXXXXXX.mp4
    annotations/task-0018/episode_XXXXXXXX.json
```

主 parquet 保留原始 BEHAVIOR/LeRobot 欄位，例如：

```text
index
episode_index
task_index
timestamp
observation.state
observation.cam_rel_poses
observation.task_info
action
```

並新增：

```text
a2c2.base_action_chunk  # fixed-size float32 [32, 23]
a2c2.valid_action_mask  # fixed-size bool [32]
a2c2.policy_infer_ms    # float32
```

latent 不塞進主 parquet，而是 sidecar：

```text
latent/data/task-0018/episode_XXXXXXXX.parquet
```

每個 latent parquet 只存一欄：

```text
a2c2.base_policy_z  # fixed-size float32 [latent_dim]
```

主 parquet 和 latent parquet 以 episode 和 row order 對齊。主 parquet 第 `i`
row 的 `a2c2.base_action_chunk[i]` 對應 latent parquet 第 `i` row 的
`a2c2.base_policy_z[i]`。

## 訓練對齊方式

訓練 correction head 時，對每個 source row `source_idx` 和 chunk offset `k`：

```text
valid         = main[source_idx]["a2c2.valid_action_mask"][k]
z             = latent[source_idx]["a2c2.base_policy_z"]
base_action   = main[source_idx]["a2c2.base_action_chunk"][k]
target_idx    = source_idx + k
expert_action = main[target_idx]["action"]
delta_target  = expert_action - base_action
```

只有 `valid == True` 的 pair 才應該用來訓練，因為 episode 尾端的 chunk 可能
超出 demonstration 長度。

## 重新建立資料

一般使用者不需要重新建立資料；直接下載即可。若要從 BEHAVIOR-1K source demos
和 OpenPI/COMET checkpoint 重建，使用 `--build`：

```bash
python a2c2_create_dataset.py \
  --build \
  --source-root /root/b1k/BEHAVIOR-1K/OmniGibson/datasets/2025-challenge-demos \
  --openpi-root /root/b1k/openpi-comet \
  --checkpoint-dir /root/b1k/openpi-comet/checkpoints/pi05-b1kpt50-cs32 \
  --output-root /root/b1k/a2c2_dataset/tidying_bedroom_pi05-b1kpt50-cs32_h32_v1 \
  --config-name pi05_b1k-base \
  --task-name tidying_bedroom \
  --cache-seed 42
```

重建模式預設支援 resume：

- 每個 episode 的主 parquet 和 latent parquet 都用 `.tmp` 寫完後 atomic replace。
- 完成的 episode 會寫入 `meta/a2c2_progress/episode_XXXXXXXX.json`。
- 續跑時會檢查 row count 和 schema，完整的 episode 會跳過。
- `--overwrite` 會刪除 output root 後重建。

小型 schema 檢查可以使用 mock policy：

```bash
python a2c2_create_dataset.py \
  --build \
  --mock-policy \
  --max-episodes 1 \
  --max-frames-per-episode 4 \
  --output-root /tmp/a2c2_schema_check \
  --overwrite
```
